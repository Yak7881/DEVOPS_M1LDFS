## Mini-quiz (5 questions)

**1.** Où cliques-tu pour créer une nouvelle issue sur GitHub ?

**2.** Quelle est la différence entre "base" et "compare" dans une Pull Request ?

**3.** À quoi sert l'onglet **Actions** sur GitHub ?

**4.** Où se trouve le bouton vert pour merger une PR ?

**5.** Comment créer une branche depuis l'interface web sans utiliser Git en ligne de commande ?

<details>
<summary>Voir les réponses</summary>

1. Dans l'onglet **Issues**, clique sur le bouton vert **New issue**.
2. **base** = branche cible (ex : `main`). **compare** = ta branche (celle que tu veux fusionner).
3. L'onglet **Actions** affiche tous les pipelines (vérifications automatiques à chaque push ou PR).
4. Le bouton vert **"Merge pull request"** en bas de la page de la PR.
5. Clique sur le sélecteur de branche (défaut : `main`), puis **New branch**, tape le nom et crée. Ou : en éditant un fichier, coche "Create a new branch" avant de commit.

</details>

## Mini-tâche

**Objectif :** Créer une issue **et** une Pull Request sur ton propre fork de ce repo.

**Étapes :**

1. **Fork** ce repo devops-rooms si ce n'est pas déjà fait
2. **Créer une issue** sur ton fork : décris une petite amélioration (ex : "Ajouter un exemple dans la room 04")
3. **Créer une branche** depuis la GUI (ex : `feature/ma-contribution`)
4. **Éditer un fichier** (ex : ajouter une ligne dans un README) et commit
5. **Ouvrir une PR** : compare ta branche avec `main` de ton fork

**Validation :** Tu as une issue ouverte et une PR ouverte sur ton fork. Pas besoin de merger, l'important est de faire le parcours complet.
