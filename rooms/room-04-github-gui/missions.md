# Missions — Room 04 (GitHub GUI)

5 missions, 3 points chacune. Fais-les dans l'ordre.

---

## Mission 1 — Créer une issue sur un repo (3 pts)

**Objectif :** Créer une issue via l'interface web de GitHub.

**Étapes (avec la GUI) :**

1. Va sur un repo GitHub (ex : ce repo, ou un fork)
2. Clique sur **Issues** dans le menu de gauche
3. Clique sur **New issue** (bouton vert)
4. Remplis le **titre** : ex. "Bug : typo dans le README"
5. Remplis la **description** : décris le problème en 2-3 phrases
6. Clique sur **Submit new issue**

**Validation :** Tu as créé une issue réelle. Note l'URL ou fais une capture d'écran.

---

## Mission 2 — Créer une branche depuis l'interface web (3 pts)

**Objectif :** Créer une branche sans utiliser la ligne de commande.

**Étapes (avec la GUI) :**

1. Va sur un repo GitHub
2. Clique sur le **sélecteur de branche** (affiche "main" ou le nom actuel)
3. Clique sur **New branch** ou tape un nom dans la zone de recherche
4. Nomme ta branche : ex. `fix/typo-readme` ou `feature/ma-branche`
5. Valide la création

**Alternative :** Ouvre un fichier → clique sur l'icône crayon → avant de commit, coche "Create a new branch" et donne un nom.

**Validation :** Tu as une branche créée. Vérifie qu'elle apparaît dans le sélecteur.

---

## Mission 3 — Éditer un fichier directement sur GitHub et commit (3 pts)

**Objectif :** Modifier un fichier et enregistrer le changement via le navigateur.

**Étapes (avec la GUI) :**

1. Va sur un repo (idéalement ton fork)
2. Ouvre un fichier (ex : README.md)
3. Clique sur l'icône **crayon** (Edit this file)
4. Modifie le contenu (ex : ajoute une ligne, corrige une faute)
5. En bas : écris un message de commit (ex : "fix: corriger une typo")
6. Choisis : "Commit directly to main" ou "Create a new branch"
7. Clique sur **Commit changes**

**Validation :** Le fichier est modifié. Tu vois ton commit dans l'historique.

---

## Mission 4 — Ouvrir une Pull Request depuis la GUI (3 pts)

**Objectif :** Créer une PR sans toucher à la ligne de commande.

**Étapes (avec la GUI) :**

1. Va sur un repo GitHub
2. Clique sur **Pull requests** dans le menu
3. Clique sur **New pull request**
4. Choisis **base** (ex : `main`) et **compare** (ta branche)
5. Vérifie les changements affichés (diff)
6. Clique sur **Create pull request**
7. Remplis le titre et la description
8. Clique sur **Create pull request** (bouton vert)

**Validation :** Tu as ouvert une PR. Tu peux la partager ou la merger si tu as les droits.

---

## Mission 5 — Consulter l'onglet Actions et lire un log de pipeline (3 pts)

**Objectif :** Trouver l'onglet Actions et comprendre un pipeline.

**Étapes (avec la GUI) :**

1. Va sur un repo qui a des pipelines (ex : ce repo devops-rooms)
2. Clique sur **Actions** dans le menu du repo
3. Trouve une exécution (run) de pipeline — verte ou rouge
4. Clique sur une exécution pour l'ouvrir
5. Clique sur un **job** (ex : "build" ou "test")
6. Déroule une **étape** (step) et lis le log

**Validation :** Tu as compris où se trouvent les logs et ce qu'ils affichent (commandes, messages, erreurs).
