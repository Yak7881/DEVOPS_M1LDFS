# Room 01 — Contrôle de version (Git)

Bienvenue dans ta première vraie room technique ! Tu vas apprendre à **sauvegarder et suivre ton code avec Git**.

---

## Ce que tu apprends

Comment sauvegarder et suivre ton code avec Git.

---

## Pourquoi c'est utile ?

Pour ne jamais perdre ton travail et pouvoir revenir en arrière à tout moment.

---

## Exemple concret

Tu as modifié un fichier et tout est cassé. Sans Git, tu es bloqué. **Avec Git**, tu peux revenir à la version qui marchait en une commande.

---

## Les termes à connaître

| Terme | Définition courte |
|-------|-------------------|
| **Repository (dépôt)** | Un dossier de projet géré par Git. Il contient tout le code et l'historique des modifications. |
| **Commit** | Une sauvegarde de tes modifications dans Git. Chaque commit a un message qui décrit ce qui a changé. |
| **Branch (branche)** | Une copie parallèle de ton code. Tu travailles dessus sans toucher au code principal. |
| **Push** | Envoyer tes commits locaux vers le dépôt distant (ex : GitHub). |
| **Pull** | Récupérer les modifications du dépôt distant sur ton ordinateur. |

Pour les définitions complètes : [glossaire](../../docs/glossary.md).
