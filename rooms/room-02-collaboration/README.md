# Room 02 — Collaboration

Tu sais utiliser Git en local. Maintenant, tu vas apprendre à **travailler avec d'autres sur le même projet**.

---

## Ce que tu apprends

Comment travailler avec d'autres sur le même projet.

---

## Pourquoi c'est utile ?

En vrai, les projets impliquent toujours plusieurs personnes. Il faut savoir collaborer sans se marcher sur les pieds.

---

## Exemple concret

Deux développeurs travaillent sur le même projet. L'un ajoute un bouton, l'autre corrige un bug. **Avec Git**, chacun travaille sur sa branche. Ils n'écrasent pas le travail de l'autre. Ensuite, ils fusionnent proprement.

---

## Les termes à connaître

| Terme | Définition courte |
|-------|-------------------|
| **Clone** | Copier un dépôt distant (sur GitHub) sur ton ordinateur. |
| **Fork** | Copier le dépôt de quelqu'un d'autre sur ton propre compte GitHub. Tu peux le modifier sans toucher à l'original. |
| **Merge (fusionner)** | Combiner les modifications d'une branche dans une autre (souvent dans main). |
| **Conflit** | Quand deux personnes ont modifié la même ligne. Git ne sait pas quelle version garder. Tu dois choisir manuellement. |
| **Pull Request (PR)** | Une demande pour fusionner ta branche dans la branche principale. C'est aussi l'endroit où l'équipe relit ton code. |

Pour les définitions complètes : [glossaire](../../docs/glossary.md).
