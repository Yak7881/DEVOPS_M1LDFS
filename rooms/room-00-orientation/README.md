## Qu'est-ce que le DevOps ?

**DevOps**, c'est un ensemble de pratiques pour que les développeurs (ceux qui écrivent le code) et les opérationnels (ceux qui font tourner les serveurs) travaillent **ensemble**, plus vite, avec moins d'erreurs.

En résumé : **écrire du code → le tester → le livrer → le surveiller**, le tout de manière fluide et automatisée.

---

## Le système des rooms

Ce parcours est divisé en **rooms** (salles). Chaque room = un concept.

| Élément | Description |
|--------|-------------|
| **README** | La leçon. Lis toujours en premier. |
| **Missions** | Des exercices pratiques à faire (quand la room en a) |
| **Checkpoint** | Un mini-quiz + une mini-tâche pour valider tes acquis |

Les rooms sont numérotées. Suis-les dans l'ordre (00 → 01 → 02…).
