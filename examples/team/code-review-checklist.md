# Checklist — Revue de code

En tant que reviewer, utilise cette checklist pour relire une PR.

---

## Fonctionnel

- [ ] Le code fait ce qui est décrit dans la PR
- [ ] Les cas limites sont gérés
- [ ] Pas de régression évidente

---

## Qualité

- [ ] Le code est lisible et bien structuré
- [ ] Pas de code dupliqué inutile
- [ ] Les noms de variables/fonctions sont clairs

---

## Sécurité & performance

- [ ] Pas de données sensibles exposées
- [ ] Pas de risque évident (injection, etc.)

---

## Feedback

- **Approve** : tout est bon
- **Comment** : question ou suggestion
- **Request changes** : correction nécessaire
